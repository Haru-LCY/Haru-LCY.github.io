为什么要用对数（log）计算损失，而不直接用概率？

使用对数计算损失的主要原因有 数学性质、优化效果、概率解释 等几个方面。

1. 概率 vs. 对数损失

如果直接用概率作为损失：
[
L_i = 1 - P(y_i)
]
这个损失的变化幅度较小，优化效果不好。

但是，使用对数损失（交叉熵损失）：
[
L_i = -\log P(y_i)
]
它对 小概率的错误预测 给予更大的惩罚，使得模型更容易学习到正确的分类。

2. 对数损失的优点

(1) 让小概率预测的损失更大

如果模型对正确类别的预测概率很低，例如 ( P(y_i) = 0.01 )，那么：
[
L_i = -\log(0.01) = 4.61
]
损失较大，梯度也较大，使得优化过程更快收敛。

如果我们直接用概率损失：
[
L_i = 1 - 0.01 = 0.99
]
变化不够明显，优化过程可能会更慢。

(2) 使损失函数具有凸性，方便优化

在优化（如梯度下降）时，我们希望损失函数是 光滑、凸的，这样优化算法可以稳定地找到最优解。

交叉熵损失是一个 凸函数，具有良好的优化性质。相比之下，直接使用概率作为损失，优化过程可能会不稳定。

(3) 与最大似然估计（MLE）一致

在统计学中，训练模型的目标是 最大化正确类别的概率，这叫做 最大似然估计（MLE）：
[
\max \prod{i} P(y_i)
]
最大化概率 等价于 最小化交叉熵损失：
[
\min -\sum{i} \log P(y_i)
]
所以，使用对数损失可以让 Softmax 分类器与概率论中的最大似然方法保持一致。

(4) 概率直接作为损失可能会导致梯度消失

如果直接使用概率作为损失：
[
L_i = 1 - P(y_i)
]
当 ( P(y_i) ) 接近 1 时，损失变化非常小，梯度也会变小，这样参数更新速度会减慢，可能导致 梯度消失 问题。

使用对数：
[
L_i = -\log P(y_i)
]
即使 ( P(y_i) ) 很接近 1，梯度依然不会变得太小，有助于优化过程的稳定性。

结论

✅ 对数损失的优点：

  • 小概率预测损失大，训练更有效。
  • 使损失函数光滑、凸，优化更稳定。
  • 与最大似然估计一致，数学上合理。
  • 避免梯度消失，提高训练效果。

所以，我们使用 对数损失（交叉熵） 而不是直接使用概率作为损失。
